# ✅ Чеклист тестування після застосування міграцій

Після застосування всіх міграцій БД, виконайте ці перевірки:

## 🗄️ **Перевірка бази даних**

### 1. Перевірка колонок scope
- [ ] Таблиця `conversations` має колонку `scope` зі значеннями: `'community'`, `'project'`, `'personal'`
- [ ] Таблиця `agents` має колонку `scope` зі значеннями: `'community'`, `'personal'`
- [ ] Таблиця `files` має колонку `scope` зі значеннями: `'community'`, `'project'`, `'personal'`
- [ ] Таблиця `user_integrations` має колонку `scope` зі значеннями: `'team'`, `'personal'`

### 2. Перевірка нових таблиць
- [ ] Таблиця `user_integrations` існує та має всі необхідні колонки
- [ ] Таблиця `push_notification_settings` існує та має всі необхідні колонки

### 3. Перевірка RLS політик
- [ ] RLS увімкнено для всіх нових таблиць
- [ ] Політики доступу працюють коректно

**SQL для перевірки:**
```sql
-- Перевірка колонок
SELECT column_name, data_type, column_default
FROM information_schema.columns 
WHERE table_name = 'conversations' AND column_name = 'scope';

SELECT column_name, data_type, column_default
FROM information_schema.columns 
WHERE table_name = 'agents' AND column_name = 'scope';

SELECT column_name, data_type, column_default
FROM information_schema.columns 
WHERE table_name = 'files' AND column_name = 'scope';

-- Перевірка таблиць
SELECT table_name 
FROM information_schema.tables 
WHERE table_schema = 'public' 
AND table_name IN ('user_integrations', 'push_notification_settings');

-- Перевірка RLS
SELECT tablename, rowsecurity 
FROM pg_tables 
WHERE schemaname = 'public' 
AND tablename IN ('user_integrations', 'push_notification_settings');
```

## 🎨 **Функціональне тестування Frontend**

### 4. Тестування чатів (Conversations)
- [ ] Відкривається список чатів без помилок
- [ ] Створення нового чату працює
- [ ] Фільтрація чатів по scope (community/project/personal) працює
- [ ] Відправка повідомлень в чатах працює
- [ ] Головний агент (Dify) підключається до community та project чатів
- [ ] Головний агент НЕ підключається до personal чатів (перевірити повідомлення про недоступність)

### 5. Тестування голосового вводу
- [ ] Кнопка запису голосу працює (onClick handler)
- [ ] Запис починається при натисканні
- [ ] Запис зупиняється автоматично після 2.5 секунд мовчання
- [ ] Запис зупиняється вручну при повторному натисканні
- [ ] Після автостопу повідомлення відправляється автоматично
- [ ] Після ручної зупинки текст додається в поле вводу
- [ ] Розпізнавання мови працює (STT API)
- [ ] Помилки обробляються коректно (показуються toast-повідомлення)

### 6. Тестування агентів (Agents)
- [ ] Список агентів відображається
- [ ] Фільтрація агентів по scope (community/personal) працює
- [ ] Створення нового агента працює
- [ ] Вибір scope при створенні агента працює
- [ ] Спільний агент (community scope) відображається для всіх
- [ ] Приватні агенти (personal scope) відображаються тільки для власника

### 7. Тестування інтеграцій (Integrations)
- [ ] Сторінка інтеграцій відкривається без помилок
- [ ] Список інтеграцій відображається
- [ ] Фільтрація по scope (team/personal) працює
- [ ] Створення нової інтеграції працює
- [ ] Вибір scope (team/personal) при створенні працює
- [ ] Підключення інтеграції працює
- [ ] Відключення інтеграції працює
- [ ] Налаштування інтеграцій зберігаються

### 8. Тестування бази знань (Knowledge Base)
- [ ] Список файлів відображається
- [ ] Фільтрація по scope (community/project/personal) працює
- [ ] Завантаження файлів працює
- [ ] Файли зберігаються з правильним scope
- [ ] Створення папок працює
- [ ] Навігація по папках працює

### 9. Тестування push-сповіщень
- [ ] Сторінка налаштувань відкривається
- [ ] Налаштування push-сповіщень відображаються
- [ ] Увімкнення/вимкнення новинних сповіщень працює
- [ ] Вибір чатів для сповіщень працює
- [ ] Налаштування зберігаються в БД
- [ ] Push-сповіщення надсилаються (якщо налаштовано)

### 10. Тестування новинної стрічки (News Feed)
- [ ] Новинна стрічка відображається
- [ ] Відправка новини працює
- [ ] Відображення профілів користувачів працює (з email)
- [ ] Агент відповідає на згадку @ЖОС
