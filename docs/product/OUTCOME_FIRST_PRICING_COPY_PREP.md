## Purpose

Prepare pricing-page language that sells the result of a MicroDAO, not only the
feature list.

Current risk:

```text
Users see RAG, chats, tasks, and invitations, but may not understand why that is
worth 2, 5, 15, or 30 DAAR per month.
```

Target outcome:

```text
Users understand that DAARION MicroDAO reduces coordination chaos, preserves
community memory, accelerates onboarding, turns conversations into action, and
gives the leader an AI operator for the community.
```

## Core Copy Principle

Lead with the operating result.

Then show the features as proof.

```text
Outcome -> proof -> limit -> next action
```

Avoid this order:

```text
Feature list -> technical term -> price
```

Prefer this order:

```text
Problem solved -> community size -> operating result -> included tools
```

## Boundary Copy

### Connect Device Boundary

Use near pricing or advanced plan notes:

```text
Connect Device is part of the DAARION path, but you can enter your MicroDAO
Dashboard before a device is connected.
```

### Local Agent Runtime Boundary

```text
Local Agent activation is a future device-powered capability. It should not be
presented as included until the runtime path is implemented and validated.
```

### Worker Node Boundary

```text
Worker Node is a separate advanced/operator program. It is not included in
normal MicroDAO plans.
```

### Organization Boundary

```text
Organization plans are custom. They should not look like a normal self-serve
checkout tier until contracts, support, and infrastructure are ready.
```

## FAQ Copy Draft

### Why does a MicroDAO cost money?

```text
Because DAARION keeps more than a chat open. A MicroDAO preserves community
memory, supports AI-assisted coordination, turns conversations into tasks, and
helps new members understand the workspace faster.
```

### Why do plans scale by members?

```text
More members create more conversations, more decisions, more onboarding, more
memory, and more agent work. Plans scale with that coordination load.
```

### Is Connect Device required?

```text
No. You can use the MicroDAO Dashboard first. Connect Device becomes useful when
you need local or device-powered DAARION capabilities.
```

### Is Worker Node included?

```text
No. Worker Node is a separate advanced/operator program with its own trust and
technical requirements.
```

### Is Founding pricing permanent?

```text
Yes. The first 10 approved MicroDAOs keep the 2 DAAR/month founding price while
the MicroDAO remains active and policy-compliant.
```
